Ce projet avait pour but de relier un serveur linux ou l'on va récolter les logs d'un pc client à travers le service Rsyslog. Les logs sont stockés en fonction de la configuration que l'on à mis dans les fichiers du service du serveur et du client.

Pour ce faire, il a fallu que l’on modifie le fichier /etc/rsyslog.conf du serveur et du client afin de rajouter des lignes de configuration.
Coté serveur : En premier lieu j’ai décommenté les lignes “module(load=”imudp”) et input(type=”imudp” port=”514”” pour ouvrir le port 514 utile à rsyslog.

Coté client: Si jamais on ne voulait diriger que certains log sur le serveur comme les erreur de mail ou kernel nous avons à disposition des modules intégrés à rsyslog.

auth : Utilisé pour des évènements concernant la sécurité ou l'authentification à travers des applications d'accès (type SSH)
authpriv : Utilisé pour les messages relatifs au contrôle d'accès
daemon : Utilisé par les différents processus systèmes et d'application
kern : Utilisé pour les messages concernant le kernel
mail : Utilisé pour les évènements des services mail
user : module par défaut quand aucune n'est spécifiée
local : Utilisé pour les messages du boot
* : Désigne toutes les modules
none : Désigne aucun module

En plus de précision, nous retrouvons pour chaque module un niveau de gravité

Emerg : Urgence, système inutilisable
Alerte : alerte Intervention immédiate nécessaire
Crit : Erreur système critique
Err : Erreur de fonctionnement
Warning : Avertissement
Notice : Évènement normaux devant être signalé
Info : Pour information simple
Debug : Message de débogage

Sachant que l’objectif était de tout regrouper dans un seul fichier la seul chose à faire était de rajouter une ligne à la fin du fichier rsyslog.conf -> *.* @IP_SRV:514
